# Release Notes

This section contains release notes for each version of the Construction Quality Management application. Release notes document all changes, new features, improvements, bug fixes, and any breaking changes introduced in each release.

## Release Notes Process

1. **Before release** — Copy [TEMPLATE.md](./TEMPLATE.md) and rename it to the version number (e.g., `v1.0.0.md`)
2. **Categorize changes** — Place each change in the appropriate section (New Features, Improvements, Bug Fixes, etc.)
3. **Document breaking changes** — Any change that requires action from users or developers must be listed under Breaking Changes with clear migration steps
4. **Update this index** — Add the new release to the Release History table above
5. **Review** — Ensure all changes from the milestone/sprint are captured before publishing

## Versioning

This project follows [Semantic Versioning](https://semver.org/):

- **MAJOR** (x.0.0) — Breaking changes that require migration
- **MINOR** (0.x.0) — New features added in a backward-compatible manner
- **PATCH** (0.0.x) — Bug fixes and minor improvements

Use the [Release Notes Template](./TEMPLATE.md) as the starting point for each new release document. It provides a consistent structure with all required sections.
